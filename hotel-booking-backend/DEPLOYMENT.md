# Deployment Guide

## Backend: Deploy to Render

1. **Push code to GitHub:**
   ```bash
   git init
   git add .
   git commit -m "Initial backend setup"
   git remote add origin YOUR_GITHUB_REPO_URL
   git push -u origin main
   ```

2. **Deploy on Render:**
   - Go to https://render.com (sign in with GitHub)
   - Click "New +" → "Web Service"
   - Connect your GitHub repo
   - Render auto-detects `render.yaml`
   - Click "Create Web Service"

3. **Set Environment Variables:**
   In Render dashboard, add:
   - `MONGODB_URI` → Your MongoDB Atlas connection string
   - `TELEGRAM_BOT_TOKEN` → From BotFather
   - `OWNER_CHAT_ID` → Your Telegram chat ID
   - `FRONTEND_URL` → `https://your-frontend.vercel.app` (add after frontend is deployed)
   - `JWT_SECRET` → Auto-generated by Render
   - `ADMIN_SEED_PASSWORD` → Your admin seed password
   - `STAFF_SEED_PASSWORD` → Your staff seed password

4. **Copy your backend URL:**
   After deployment, Render gives you a URL like:
   ```
   https://hotel-booking-api.onrender.com
   ```
   Save this - the frontend needs it!

## Frontend: Deploy to Vercel

*This will be done by Antigravity, but here's the flow:*

1. **Create `.env.production` in frontend:**
   ```
   VITE_API_URL=https://hotel-booking-api.onrender.com
   ```

2. **Deploy:**
   ```bash
   cd hotel-booking-frontend
   vercel
   ```

3. **Update backend CORS:**
   After frontend deploys, add the Vercel URL to backend's `FRONTEND_URL` env var on Render.

## MongoDB Atlas Setup (if needed)

1. Go to https://www.mongodb.com/cloud/atlas
2. Create free cluster (M0 tier)
3. Create database user
4. Whitelist IP: `0.0.0.0/0` (allow all - needed for Render)
5. Get connection string:
   ```
   mongodb+srv://username:password@cluster.mongodb.net/hotel-booking
   ```
6. Add this to Render's `MONGODB_URI` env var

## Testing Production

```bash
# Test health check
curl https://hotel-booking-api.onrender.com/health

# Test login
curl -X POST https://hotel-booking-api.onrender.com/api/auth/login \
  -H "Content-Type: application/json" \
   -d '{"email":"admin@hotel.com","password":"<ADMIN_SEED_PASSWORD>"}'
```

## Important Notes

- **First request may be slow** (Render free tier spins down after inactivity)
- **Run seed script after first deploy** to create admin user
- **Telegram bot works immediately** once env vars are set
